  <name>Task 1: Register StarletteHTTPException handler in exceptions.py</name>
  <files>app/core/exceptions.py</files>
  <action>
The problem: When FastAPI cannot match a route, Starlette internally raises `starlette.exceptions.HTTPException(status_code=404)`. Starlette's own ExceptionMiddleware catches this BEFORE our generic `Exception` handler fires, returning the bare `{"detail": "Not Found"}` shape. We must register a dedicated handler for this exact exception type.

Changes to make to `app/core/exceptions.py`:

1. Add import after the existing FastAPI/Starlette imports:
   ```python
   from starlette.exceptions import HTTPException as StarletteHTTPException
   ```

2. Inside `register_exception_handlers(app)`, add the following handler AFTER the `RequestValidationError` handler and BEFORE the generic `Exception` handler:
   ```python
   @app.exception_handler(StarletteHTTPException)
   async def starlette_http_handler(request: Request, exc: StarletteHTTPException):
       status_code_map = {
           400: "BAD_REQUEST",
           401: "UNAUTHORIZED",
           403: "FORBIDDEN",
           404: "NOT_FOUND",
           405: "METHOD_NOT_ALLOWED",
           409: "CONFLICT",
           422: "UNPROCESSABLE_ENTITY",
           500: "INTERNAL_ERROR",
       }
       code = status_code_map.get(exc.status_code, "HTTP_ERROR")
       return JSONResponse(
           status_code=exc.status_code,
           content={
               "detail": exc.detail or code,
               "code": code,
               "statusCode": exc.status_code,
           },
       )
   ```

Do NOT move or modify any existing handler. Only add the new import and the new handler function in the position described.
